A Distributional Comparison of Human and VARC Failures on ARC}
\author{Jing Yu \and Huijing Cong \and Wanyi Zhou}

\begin{abstract}
The Abstraction and Reasoning Corpus (ARC) remains one of the few benchmarks
on which strong neural systems still trail human performance. Most prior
analyses report task-level accuracy, treating each task as a binary
pass/fail event. We argue that this hides a more fundamental difference in
\emph{how} the two agents fail. Using the official VARC ViT model's 510
multi-view predictions per task and 946 participants from H-ARC, we treat
each side's predictions as a sample from an answer distribution and compare
their entropy, top-1 vote share, and the number of distinct candidate
grids. On the 111 tasks where humans succeed but VARC fails, humans
converge on $\approx 3.6$ distinct answers (top-1 share 74\%, entropy
$0.80$); on the same tasks VARC scatters across $\approx 264$ distinct
grids (top-1 share 13\%, entropy $4.72$). The correlation between human
and VARC entropy is only $0.21$. We argue this signature reflects a
\emph{failure-to-induce}: rather than picking among a small hypothesis
set as humans do, the model produces hundreds of pixel-space candidates,
suggesting it never localizes a rule.
\end{abstract}

\section{Introduction}

The Abstraction and Reasoning Corpus~\cite{chollet2019measure} was
designed as a deliberate counterpoint to large-scale supervised
benchmarks: each task supplies only a handful of input--output examples
and asks the solver to induce a transformation rule that generalizes to
held-out inputs. Modern systems, including the VARC family of vision
transformers~\cite{varc}, have closed much of the gap to human
performance, but a substantial residual remains. The natural next
question is not how much remains, but \emph{what kind of failure
accounts for it}.

Prior work largely reports per-task accuracy or pass@$k$ scoring on
ARC, which collapses each task to a single bit and obscures the
underlying behavior. The H-ARC dataset~\cite{harc} provides the
complementary human-side signal: multiple participants attempting each
task, with intermediate and final grids logged. Concurrently, VARC's
inference procedure already provides an analogous signal on the model
side: each test input is rendered through 510 augmented \emph{views},
and the final prediction is a majority vote over the 510 view-level
outputs. The 510-vote distribution is not normally examined directly, but
it is exactly the distribution we would want if we were asking what
hypothesis the model entertains.

We exploit this symmetry: $\sim 10$ human responses per task and $510$
view predictions per task can both be summarized as a distribution over
answer grids. We ask whether the \emph{shape} of that distribution
distinguishes the two agents on the tasks that matter most --- those
where humans succeed and the model fails.

\paragraph{Contributions.}
\begin{itemize}
  \item We introduce a passive input-intervention analysis that reuses
        VARC's existing 510-view predictions to characterize model
        confidence at task level, with no additional inference.
  \item We define matched answer-distribution metrics for human and
        model and compare them on H-ARC's 400 evaluation tasks.
  \item We identify a qualitative asymmetry: on tasks humans solve
        and VARC misses, humans concentrate on a handful of candidate
        grids while the model scatters across hundreds. We argue this is
        a \emph{failure-to-induce} signature, not a fine-grained error.
\end{itemize}

\section{Methods}

\subsection{Data}

We use the 400-task ARC-AGI evaluation set as the shared benchmark. For
the model side we use VARC's official ARC-1\_ViT predictions, specifically
the \texttt{attempt\_0} folder, which contains 510 view-level predicted
grids per test input. For the human side we use H-ARC's
\texttt{summary\_data.csv}, which contains 7820 attempt-level submissions
from 946 participants. We restrict to \texttt{task\_type == evaluation}
throughout and take each participant's \emph{last} attempt as their
final answer for that task. Table~\ref{tab:data} summarizes the data.

\subsection{Correctness scoring}

For VARC we follow the official plain-ViT scoring: within
\texttt{attempt\_0}'s 510 views per test input, count grid-level votes
and retain the top-2 most-frequent grids; the task is correct if either
of those two matches the ground truth (pass@2). For human correctness
on a task we report \texttt{human\_accuracy}, the fraction of
participants whose final answer matches ground truth. To form the four
quadrants in Section~\ref{sec:results}, we declare the human side
``correct'' on a task when \texttt{human\_accuracy} $> 0.5$.

\subsection{Error typing}

For diagnostic purposes (independent of the pass@2 correctness
calculation) we classify each predicted grid against ground truth into
one of four categories:
\begin{itemize}
  \item \texttt{correct}: exact match.
  \item \texttt{wrong\_size}: row or column count differs from ground
        truth.
  \item \texttt{close\_miss}: shape matches and cell-level accuracy
        $\geq 0.8$.
  \item \texttt{wrong\_content}: shape matches but cell-level accuracy
        $< 0.8$.
\end{itemize}
For VARC we use the top-1 voted grid for error typing, mirroring how the
single most-likely model output would be inspected.

\subsection{Answer-distribution metrics}

The core methodological move is to treat the 510 view-level VARC
predictions and the $\sim 10$ human final answers as samples from
parallel \emph{answer distributions} on each task. For each task and
each agent we compute, over the empirical distribution of distinct grids
$\{g_i\}$ with vote counts $n_i$ (total $N$):
\begin{align*}
\text{top-1 share} &= \frac{\max_i n_i}{N},\\
\text{top-1/top-2 gap} &= \frac{n_{(1)} - n_{(2)}}{N},\\
H &= -\sum_i \frac{n_i}{N} \log \frac{n_i}{N}\quad\text{(Shannon entropy, nats)},\\
n_{\mathrm{unique}} &= |\{g_i\}|.
\end{align*}
For the model these summarize the 510-view output distribution; for
humans they summarize the per-task distribution of last-attempt grids.
Because both sides are scored on the same units (distinct grids,
proportions of agents), the metrics are directly comparable.

\subsection{Quadrant taxonomy}
\label{sec:quadrants}

We partition the 400 tasks by joint correctness:
\begin{itemize}
  \item \texttt{both\_correct} ($n=173$),
  \item \texttt{human\_only} ($n=111$): human majority correct, VARC
        pass@2 wrong,
  \item \texttt{varc\_only} ($n=48$): VARC correct, human majority
        wrong,
  \item \texttt{both\_wrong} ($n=68$).
\end{itemize}
The \texttt{human\_only} group is the focus of our analysis: these are
the tasks on which the model's failure is least excusable, since the
problem demonstrably admits a solution that ordinary participants find.

quadrants (Table~\ref{tab:varc_view}), we find a sharp split: when VARC
gets the task right (whether or not humans do), its 510 views agree on
a small handful of grids; when VARC gets the task wrong, the 510 views
scatter across hundreds of distinct outputs.

Two patterns are worth highlighting. First, on \texttt{human\_only}
tasks, the unique-grid count differs by nearly two orders of magnitude
($\approx 3.6$ vs $\approx 264$). Humans don't merely outperform; they
do so by entertaining a different size of hypothesis set. Second,
\texttt{both\_wrong} tasks show humans \emph{also} disperse (entropy
$1.77$, $\approx 7$ distinct grids) --- but still an order of magnitude
less than VARC's $\approx 281$ grids. Even on tasks no one can solve,
the human candidate set is bounded; the model's is not.

The Pearson correlation between per-task VARC entropy and human entropy
is $r = 0.21$ ($p = 3.1 \times 10^{-5}$, $n = 400$). This is non-trivial
but small: tasks that confuse humans and tasks that confuse the model are
largely different tasks. The two agents have different difficulty profiles,
not a shared one with one agent uniformly weaker.

The entropy gap on \texttt{human\_only} tasks is statistically robust.
A Mann-Whitney $U$ test comparing VARC vote entropy on \texttt{human\_only}
tasks against human answer entropy on the same tasks yields
$U = 12291$, $p < 10^{-30}$ (one-sided), with a large effect size
(Cohen's $d = 1.92$ relative to \texttt{both\_correct} tasks).
Critically, VARC entropy does \emph{not} differ significantly between
\texttt{human\_only} and \texttt{both\_wrong} tasks ($U = 3577$, $p = 0.56$),
confirming that the model's confusion does not track whether the task is
in principle solvable. Figure~\ref{fig:entropy_boxplot} summarises the
full per-quadrant distributions.

\subsection{Extreme cases: tasks with maximal entropy gap}

\emph{entropy gap}
\[
  \Delta H_i \;=\; H_{\mathrm{VARC},i} - H_{\mathrm{Human},i}
\]

\texttt{attempt} folders (2\,040 VARC predictions per task) to obtain a
denser sample from the model's prediction distribution; the human-side
computation is unchanged. Figure~\ref{fig:entropy_gap_scatter} re-plots
the human-vs.-VARC entropy plane with the ten largest-gap tasks circled.

These three cases carry a simple implication: the large entropy values in
the \texttt{human\_only} aggregate (Table~\ref{tab:human_vs_varc}) are
not an artefact of averaging over heterogeneous tasks. On the most extreme
individual instances, the failure-to-induce pattern from
Section~\ref{sec:human_model} reaches its logical limit --- a task that is
unambiguously and universally solved by humans yet produces near-maximum
entropy in the model's output distribution.

\subsection{Qualitative illustration}

Figure~\ref{fig:qualitative} shows a concrete \texttt{human\_only} task
(id: \texttt{b457fec5}) that exemplifies the failure-to-induce signature.
The training examples reveal a clear rule: a grey diagonal stripe is
recoloured with the colours of small marker squares placed along it.
Among the 12 participants who attempted the task, all who submitted a final
answer converged on at most 3 distinct grids (entropy $0.87$), with the
majority answer exactly matching ground truth.
VARC's 510 views, by contrast, scatter across \textbf{504 distinct prediction
grids} (entropy $6.22$) --- effectively a near-uniform distribution over the
output space. The model's top-1 prediction has the right rough structure
but applies colours inconsistently, and no single prediction captures more
than a fraction of the vote. This is qualitatively different from
a model that ``almost gets it'': it is a model that has not formed any
stable hypothesis about what the rule is.

\section{Discussion and Conclusion}

\subsection{Failure-to-induce}

The strongest reading of our results is that the two agents are not
failing at the same operation. Human errors --- both on
\texttt{human\_only} (when participants disagree on a partially correct
answer) and on \texttt{both\_wrong} (when no participant succeeds) ---
sit in a small hypothesis space: at most a handful of distinct grids,
typically reflecting plausible but conflicting transformation rules.
VARC's failures, by contrast, are nearly uniform over hundreds of
candidate grids per task. The model is not picking the wrong rule from
a shortlist; there is no shortlist.

We label this signature \emph{failure-to-induce}. The model's
multi-view procedure samples augmented presentations of the same input,
and a system that had localized a transformation rule should be roughly
view-invariant on it; the rule should produce a stable answer regardless
of the augmentation. The fact that the 510 views disagree across
hundreds of distinct outputs on the model's failure tasks suggests no
such rule has been localized.

\subsection{Why VARC's own confidence isn't enough}

A natural alternative diagnostic would be to use the model's own
view-level disagreement as a confidence signal --- low entropy meaning
``model knows,'' high entropy meaning ``model doesn't know.''
Section~3.2 shows this signal is real but limited: it cleanly
separates VARC-correct from VARC-wrong, but it cannot separate
\texttt{human\_only} from \texttt{both\_wrong}. The model is equally
confused on tasks that have a human-accessible solution and on tasks
that don't. Self-reported confidence does not, on its own, identify the
diagnostically interesting failures; it requires the human-side
distribution as a baseline.

\subsection{Limitations}

Our intervention is passive: we exploit VARC's existing 510-view
sampling rather than designing controlled input edits (color
permutation, rotation, etc.). This means we can characterize the
\emph{shape} of model failure but cannot causally identify the missing
invariance. Second, we use only the plain ViT \texttt{attempt\_0}
configuration, not the ensemble setup described in the original VARC
paper; the ensemble may be sharper. Third, the human distribution per
task is built from $\approx 10$ participants, which limits the
resolution of the human entropy estimate.

\subsection{Future work}

A controlled active-intervention follow-up is the natural sequel:
applying targeted augmentations (color permutation, rotation, mirror,
scale) on the \texttt{human\_only} subset and measuring how the
view-level prediction set responds. Given that
Section~3.2 shows VARC is already maximally diffuse on these tasks, the
question is whether \emph{any} structured intervention sharpens the
distribution; if not, that itself is evidence that the model's failure
is upstream of any specific invariance. A second direction is to use
the entropy gap between humans and the model as a difficulty signal
for constructing harder ARC evaluation sets.

\subsection{Conclusion}

On the ARC tasks that ordinary humans solve and the VARC ViT model
fails, the two agents are not failing in the same way. Humans
converge: their final answers cluster on a handful of grids, with one
clearly modal. The model wanders: its 510 view-level predictions
spread across hundreds of distinct grids, with no candidate exceeding
$13\%$ of the vote. The cognitive difference is not a matter of
sharper precision; it is a matter of \emph{whether a hypothesis set has
been formed at all}.
