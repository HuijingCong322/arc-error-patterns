## 一、修复 Figure 2：统一 VARC 指标口径（`error_distribution_pass2.png`）

**改了什么**

原来的 `error_distribution.png` 里，VARC 的 "correct" 柱高约 51%，用的是 **top-1 单次预测准确率**。但 paper 正文声明 VARC 准确率为 55.3%，用的是 **pass@2**（510个view里取top-2，任一匹配即正确）。两个数字出现在同一篇 paper 里，读者会觉得矛盾。

**为什么这样改**

Paper 的核心分析（四象限划分、view entropy 指标）全部基于 pass@2 correctness。Figure 2 是对这个评分体系的直观展示，也应该用同一口径。现在新图里 VARC 的 "correct" 柱是 **55.2%**（pass@2），与正文一致。

图的 caption 也同时更新，明确写出"VARC 使用 pass@2 计分，Human 使用 per-attempt 计分"，避免歧义。

**生成方式**：`notebooks/generate_paper_figures.py` → `results/error_distribution_pass2.png`

## 二、新增 Figure 6：四象限 entropy 分布 box plot（`entropy_boxplot.png`）

**改了什么**

新增一张双面板 box plot，左侧是 VARC 510-view vote entropy，右侧是 Human answer entropy，按四个象限（both_correct / varc_only / human_only / both_wrong）分组。同时在 paper Section 3.3 的 Pearson r 段落后面新增一段文字和这张图。

**为什么这样改**

原来的 Figure 5（散点图）需要读者自己在一堆点里找规律。核心结论"VARC在human_only和both_wrong上的entropy没有显著差异"用散点图很难看出来。Box plot 可以直接显示：
- 左图：human_only（红）和 both_wrong（灰）的箱子几乎完全重叠 → 模型的困惑度和题目是否有解无关
- 右图：human_only（红）比 varc_only 和 both_wrong 的 human entropy 低得多 → 人类对这类题是有把握的

这张图让论文的核心论点（"模型失败是 failure-to-induce，不是题难"）从视觉上更直接。

**生成方式**：`notebooks/generate_paper_figures.py` → `results/entropy_boxplot.png`

## 三、新增 Figure 7：定性案例图（`qualitative_example.png`）

**改了什么**

新增一张 ARC 任务可视化图，选取 human_only 象限中 VARC entropy 最高的任务（`b457fec5`，VARC 散布到 504 种不同 grid，entropy=6.22；人类只有 3 种答案，entropy=0.87）。图中显示：
- 训练样例（输入/输出）
- 测试输入、标准答案、VARC top-1 预测（标红 ✗）
- 人类的错误答案（标红 ✗）

同时在 paper Discussion 前新增了 Section 3.4 "Qualitative illustration"，配文解释这个案例。

**为什么这样改**

Paper 全篇都是统计数字，没有任何一张实际的 ARC 格子图。对于不熟悉 ARC 的读者来说，"VARC 散布到 264 种 grid" 这个说法很抽象，不知道是什么意思。加一个具体例子可以让"failure-to-induce"这个核心概念立刻变得可理解：

> 这道题的规则很清楚（把对角线灰色条重新涂色），12个人里大多数都找到了，但 VARC 的 510 个视角预测了 504 种完全不同的格子——模型根本没有形成任何稳定的假设。

这个例子不是单独挑选的特例，而是"VARC 最困惑的 human_only 任务"，有统计依据。

**生成方式**：`notebooks/generate_paper_figures.py` → `results/qualitative_example.png`

## 四、新增统计显著性检验 + 修复数字错误

### 4a. 统计显著性（paper Section 3.3 新增段落）

**改了什么**

在 Section 3.3 末尾的 Pearson r 那句话后面，新增了一段统计检验结果：

| 检验内容 | 结果 |
|---------|------|
| Pearson r 加 p-value | r = 0.21, **p = 3.1×10⁻⁵** |
| human_only 上 VARC entropy vs Human entropy | Mann-Whitney **p < 10⁻³⁰**，单尾 |
| human_only vs both_correct（VARC entropy） | U = 17201，**p < 10⁻²⁹**，Cohen's d = 1.92 |
| human_only vs both_wrong（VARC entropy） | p = 0.56，**不显著** |

**为什么这样改**

原来的文字只报告了 r = 0.21，没有 p-value，无法判断这个相关是否有意义。对于一篇声称发现"显著差异"的 paper，缺少统计检验是学术上的明显缺口。

最重要的是第四行：human_only 和 both_wrong 在 VARC entropy 上无显著差异（p=0.56），这正是论文核心论点的统计支撑——模型的困惑程度和"题目有没有解"无关。这句话如果没有 p-value，说服力大打折扣。

详细数字保存在 `results/significance_report.txt`。

### 4b. 修复一个数字错误

**改了什么**

paper 原文："mean cell-level accuracy is **0.876** for VARC"

**实际数据算出来是 0.882**。差了 0.6 个百分点。

已在 paper.tex 第 222 行将 `0.876` 改为 `0.882`。

**为什么会有这个错误**

`varc_cell_accuracy` 列是 task-level 平均值（先在每个 test example 算 cell accuracy，再对 task 内多个 test example 取均值）。原来的数字可能是从某个中间版本的数据文件里直接读出来的，略有偏差。
